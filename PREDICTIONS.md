## Методология проверки

1. Baseline (18.04.2026) — цены зафиксированы в этом файле и через Bybit API.
2. Ежедневный снимок через `scripts/check_predictions.py` — запись в CSV.
3. Финальная оценка 02.05.2026:
   - hit_target = True, если цена коснулась цели до стопа.
   - hit_stop = True, если цена коснулась стопа до цели.
   - open = цена между стопом и целью.
4. Метрика качества прогноза:
   - **Accuracy**: % гипотез, достигших цели до стопа.
   - **Sharpe-like**: среднее (max_price / entry - 1) делим на std.
   - **Avg return (mid)**: (current - entry) / entry на момент оценки.

## Как использовать

Скрипт не требует API-ключей — работает через публичное Bybit API.

### Ежедневный снапшот (текущие цены vs entry)

```bash
python3 scripts/check_predictions.py           # просто вывести
python3 scripts/check_predictions.py --save    # + дописать строку в CSV
```

Рекомендую запускать `--save` раз в день (можно через cron), чтобы получить
временной ряд в `data/predictions_track.csv` для построения графика P&L.

### Финальный вердикт (проверка target/stop)

```bash
python3 scripts/check_predictions.py --verdict --since 2026-04-18
python3 scripts/check_predictions.py --verdict --since 2026-04-18 --until 2026-05-02
```

Подтягивает дневные high/low с Bybit за период и определяет для каждой гипотезы:
- `WIN` — цена коснулась цели до стопа.
- `LOSS` — цена коснулась стопа до цели.
- `BOTH_HIT` — оба уровня задеты (редко, но бывает при большом диапазоне дня).
- `OPEN` — цена всё ещё между стопом и целью.

Плюс проверка негативных ставок: выросли ли WLDUSDT / ARBUSDT / JUPUSDT >15%.

### Что проверить после эксперимента

1. **Accuracy WIN rate**: ≥ 4/8 long → гипотезы про нарративы сработали.
2. **Avg mid-PnL**: ожидаемо +12–18% в базовом сценарии, +25–35% в bull.
3. **Negative-bet accuracy**: если 3/3 ≤ +15% → «что избегать» предсказано верно.
4. **Корреляция с BTC**: если BTC сделал +X%, а среднее по альтам < X — нарративы
   не дали альфу (нужно менять подход).
